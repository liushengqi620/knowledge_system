# 连铸质量异常数据处理方法与标签依据

## 一、处理目标

本数据处理流程用于从原始保护渣/连铸过程数据中构建质量异常预测数据集。核心目标是获得可解释、可复现、无后验泄漏的 X/y 建模表。

任务定义为：

```text
基于连铸过程 X，预测目标质量异常 quality_label。
```

其中，X 为连铸阶段可观测过程变量，Y 为由 `品质异常代码1-5` 定义的目标质量异常标签。

## 二、数据范围与字段筛选依据

保留字段的依据：

- 保留发生在连铸阶段的过程变量，包括温度、拉速、液面、流量、氩气、塞棒、结晶器热交换、电磁搅拌、锥度和振动等。
- 保留必要的追踪字段，如 `process_time`、连铸处理号、材料跟踪号、记录识别码、`source_row_id` 和 `process_sequence_key`。
- 排除后验处置或结果侧字段作为模型输入，包括热轧封锁标志、异常处置代码、改钢原因代码、炉次等级、品质异常大类和各类标签解释字段。

## 三、主标签 Y 的选取方法

当前主标签仅由 `品质异常代码1-5` 构造，标签列为：

```text
quality_label
```

选择依据是：品质异常代码直接描述质量异常事实，而热轧封锁、异常处置代码和改钢原因代码属于后验处置或改判结果，若用于定义 Y 或进入 X，容易造成后验泄漏。

| 样本角色 | quality_label | 定义 | 是否进入主任务 |
|---|---:|---|---|
| `target_positive` | 1 | `品质异常代码1-5` 至少命中目标质量缺陷代码 | 是 |
| `clean_negative` | 0 | `品质异常代码1-5` 均为空、0、null 或 missing | 是 |
| `auxiliary_excluded` | NaN | 仅命中交接段/中包过渡或其他非目标异常 | 否，单独保留 |

## 四、目标质量缺陷大类依据

目标质量缺陷大类按照连铸工艺机理和品质异常代码含义划分。进入主 Y 的类别要求与连铸过程扰动存在明确工艺关联。

| 质量大类 | 品质异常代码 | 进入主 Y | 依据 |
|---|---|---|---|
| `heat_transfer_imbalance` | 38 | 是 | 反映结晶器传热不均风险 |
| `process_fluctuation` | 18, 39, 19, 55, 56, 83 | 是 | 反映强过程波动或过程稳定性异常 |
| `speed_stopper_flow` | 34, 51, 52, 89, 57, 85 | 是 | 对应拉速、塞棒、流量控制链路 |
| `mold_level_slag_risk` | 30, 31, 54, 32, 86, 37, 50, 29 | 是 | 对应液面波动、卷渣及结晶器液面风险 |
| `temperature_flux` | 24, 27, 69, 70, 15, 33 | 是 | 对应温度、过热度和保护剂相关风险 |
| `transition_tundish` | 94, 21, 41, 42, 43, 87, 58 | 否 | 交接段/中包过渡事件，不作为目标质量缺陷 |
| `other_quality_abnormal` | 其他异常代码 | 否 | 未明确归因，保留待专家校验 |

## 五、X 特征构造方法

X 由原始过程变量和派生过程特征组成。原始过程变量按工艺模块筛选，派生特征用于表达关键过程状态、波动强度和物理状态。

| 模块 | 主要变量/特征 | 用途 |
|---|---|---|
| 温度与中包 | `liquidus_temp`, `superheat`, `TD_avg_temp`, `TD_weight_mean`, `TD_weight_range` | 描述钢液温度状态与中包稳定性 |
| 拉速与液面 | `cast_speed_mean`, `cast_speed_range`, `mold_level_range`, `mold_level_ge7`, `mold_level_ge10` | 描述拉速波动和液面稳定性 |
| 流量与塞棒 | 上水口/上滑板流量、塞棒开口度、氩气压力与流量 | 描述浇注控制链路 |
| 结晶器状态 | `heat_exchange_mean`, `heat_exchange_ew_diff`, `heat_exchange_ns_diff`, 电磁搅拌、锥度、振动参数 | 描述传热、搅拌和结晶器设备状态 |
| 保护剂 | `cover_flux_total`, `cover_flux_ratio`, `cover_flux_zero_flag` | 描述保护剂使用状态；零用量只标记，不直接判无效 |

## 六、物理质量清洗方法与依据

V2 数据处理新增物理有效性审计层。命中物理无效规则的样本不进入主任务训练、验证和测试，而是导出到 `invalid_data_quality_rows.csv`。极端但不必然错误的状态只做标记，避免过度清洗。

| 变量 | 规则 | 处理方式 |
|---|---|---|
| `liquidus_temp` | 1450-1580 | 超出范围判为物理无效 |
| `superheat` | 0-150 | 超出范围判为物理无效 |
| `TD_avg_temp` | 1400-1650 | 超出范围判为物理无效 |
| `cast_speed_mean` | > 0 | 非正值判为物理无效 |
| `TD_weight_mean` | > 0 | 非正值判为物理无效 |
| `TD_weight_range` | 0-100 | 超出范围判为物理无效 |
| `stopper_opening_mean/max/min` | 0-300 | 超出范围判为物理无效 |
| `heat_exchange_mean` | > 0 | 非正值判为物理无效 |
| `mold_level_range` | >= 0；>100 为极端标记 | 负值无效；>100 仅标记 |

## 七、当前数据处理结果

| 指标 | 结果 |
|---|---:|
| 原始样本数 | 104316 |
| 主任务样本数 | 76981 |
| `target_positive` | 45230 |
| `clean_negative` | 31751 |
| 辅助样本数 | 23584 |
| 物理无效排除样本数 | 3751 |

主任务正负样本比例约为 58.8% / 41.2%，具备继续开展二分类预警、分层质量大类评估和根因解释实验的基础。

## 八、建模使用边界

- 主模型输入使用 `quality_traceability_dataset_v2/X_process_features.csv`。
- 主任务监督标签使用 `quality_traceability_dataset_v2/y_quality_label.csv`。
- `auxiliary_quality_events.csv` 仅用于辅助解释、误差分析和知识沉淀。
- `invalid_data_quality_rows.csv` 用于数据质量审计，不进入训练、验证和测试。
- 异常处置代码和改钢原因代码可用于后验一致性分析和专家校验，但不得作为主模型输入或主标签来源。

## 九、结论

当前数据处理主线为：连铸过程变量筛选、品质异常代码定义 Y、后验字段隔离、物理有效性清洗和建模数据导出。该处理方式能够降低标签泄漏风险，提高数据可解释性，并为后续质量异常预测与根因解释模型提供稳定的数据基础。
