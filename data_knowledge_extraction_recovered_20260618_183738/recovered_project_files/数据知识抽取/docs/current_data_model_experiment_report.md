# 连铸过程工艺故障预测：数据处理、模型建立与实验结果报告

## 1. 任务背景与目标

本阶段工作的目标是基于现有钢铁生产数据，构建一个面向连铸过程的工艺故障预测模型。与直接预测最终质量异常代码不同，当前任务首先聚焦连铸环节，将连铸过程变量整理成监督学习数据集，并构造连铸过程工艺异常标签 `casting_process_fault_label`。因此，当前模型的任务可以定义为：

```text
输入 X：连铸过程生产工艺参数
输出 y：casting_process_fault_label
目标：预测当前样本是否处于连铸过程工艺异常状态
```

这样做的目的，是先将问题从“全流程质量缺陷预测”收敛到“连铸阶段异常识别”。由于原始品质异常代码可能同时受到连铸、热轧、冷却、检测等多个环节影响，直接使用质量异常代码作为标签会导致溯源目标不清晰。因此，当前阶段先建立连铸过程工艺故障标签，用于训练和验证面向连铸环节的时空图模型。

## 2. 原始数据处理流程

当前模型使用的主要数据入口为：

```text
knowledge_exports/continuous_casting_grouped_process_fault/categories/model_ready.csv
```

在加载该数据后，程序会从 `continuous_casting_all.csv` 中补充连铸上下文字段，包括：

```text
铸机号
奇偶流区分
钢级代码
炼钢材料号
材料跟踪号
熔炼号
连铸处理号
中间包号
```

这些字段主要用于线路筛选、分组分析、后续误差分析和可解释性分析。当前模型不会把所有原始字段直接送入网络，而是优先保留与连铸过程相关的工艺变量。

数据加载后，首先按照 `process_time` 进行排序，以保持真实生产过程中的时间顺序。当前实验固定在一条连铸线上进行：

```text
铸机号 = 2
奇偶流区分 = 1
```

固定线路的原因是，不同铸机和不同流区之间存在明显工况差异。如果直接混合所有产线样本，模型可能学习到产线差异，而不是连铸异常本身的工艺规律。

经过线路过滤后，当前实验数据规模为：

```text
原始 model_ready 数据：104273 条
铸机号=2 后：63369 条
奇偶流区分=1 后：31718 条
先验规则过滤后：31636 条
```

当前监督标签为：

```text
casting_process_fault_label
```

该标签是根据连铸过程变量构造的工艺故障标签，主要考虑以下异常类型：

```text
结晶器液位波动异常
拉速波动异常
中包重量波动异常
热交换不平衡
```

这意味着当前模型学习的是“连铸过程状态异常”，不是最终质量缺陷本身。

## 3. 特征构建方式

当前输入特征主要来自连铸生产过程变量，包括：

```text
中包重量相关变量
中包温度相关变量
拉速相关变量
过热度相关变量
保护渣相关变量
结晶器液位相关变量
热交换相关变量
连浇序列相关变量
```

在时间特征方面，当前模型采用轻量化时序编码：

```text
raw + diff
```

其中：

| 特征类型 | 含义 |
|---|---|
| `raw` | 当前时刻的工艺变量值 |
| `diff` | 相邻时间点之间的变化率 |

早期模型曾加入 rolling 多尺度滚动统计和 rFFT 局部频域特征，但消融实验显示，在当前规则派生标签体系下，这些复杂时序扩展容易引入噪声。因此，当前版本关闭了 rolling 和 rFFT，仅保留当前状态和短期变化趋势。

对应配置为：

```python
paper_temporal_use_derivative = True
paper_disable_temporal_rolling = True
paper_disable_frequency_temporal = True
```

## 4. 数据集划分方式

当前模型采用时间顺序切分，而不是随机切分。实际使用的切分策略为：

```text
time_ordered_positive_support
```

切分结果如下：

| 数据集 | 样本数 | 正样本数 | 负样本数 | 正样本比例 |
|---|---:|---:|---:|---:|
| Train | 20564 | 8214 | 12350 | 39.94% |
| Val | 6327 | 2806 | 3521 | 44.35% |
| Test | 4745 | 1640 | 3105 | 34.56% |

这种切分方式更接近真实生产预测场景：模型使用较早时间段的数据训练，用后续时间段的数据进行验证和测试。

## 5. 当前模型建立方式

当前主模型为：

```text
Full-Light Physics-aware Spatio-Temporal Graph Model
```

可以理解为一个“轻量化物理先验时空图网络”。它不是单纯的表格分类模型，而是将连铸工艺变量组织成图结构，再结合时间变化特征进行异常预测。

整体流程如下：

```text
model_ready.csv
→ 线路过滤
→ 时间排序
→ 工艺特征提取
→ TemporalEncoder(raw + diff)
→ 工艺先验图构建
→ 图边权精炼
→ 因果滞后图注意力编码
→ 时序特征 + 节点状态 + 弱图消息融合
→ GraphMLPRiskHead 分类
→ 验证集阈值选择
→ 测试集评估与可视化
```

模型主要由四个部分组成。

第一部分是时间编码器。它对每个工艺变量构造当前值和变化率，用于描述工艺状态及其短期变化。

第二部分是工艺图构建模块。图中的节点为工艺变量，边表示变量之间可能存在的工艺传播、控制反馈、滞后关联或质量影响关系。当前 LLM 先验已经关闭，图结构主要来自工艺规则、历史知识、人工标注和数据精炼。

第三部分是因果工艺感知图注意力编码器。它不是简单同步传播，而是采用滞后消息：

```text
h_i(t - τ_ij) → h_j(t)
```

每条边会综合边权、置信度、因果强度、滞后时间、稳定性和工况上下文，并按不同关系类型进行通道聚合。

第四部分是风险预测头 `GraphMLPRiskHead`。它是一个小型加权 MLP 分类器，使用加权 BCE 损失训练，输出工艺故障概率。

当前模型之所以称为 `Full-Light`，是因为它保留了图结构，但降低了图消息权重：

```text
fused = [temporal_features, node_states, 0.35 * graph_messages]
```

对应配置为：

```python
paper_graph_message_scale = 0.35
paper_graph_refine_alpha = 0.85
```

这表示当前模型更相信直接工艺状态和短期变化，同时让图结构作为弱先验辅助，而不是让复杂图传播主导分类结果。

## 6. 图结构更新与验证

为进一步验证图结构，本阶段新增了数据驱动图更新脚本：

```text
Scripts/update_data_refined_graph.py
```

该脚本会基于当前线路数据计算变量间的：

```text
滞后相关
正反方向支持度
边稳定性
候选边权重
候选边置信度
候选 lag
```

并输出：

```text
knowledge_exports/graph_update/data_refined_graph_prior.json
knowledge_exports/graph_update/data_refined_graph_prior.top_edges.csv
knowledge_exports/graph_update/data_refined_graph_prior.candidates.csv
```

当前数据驱动图更新得到的部分候选边包括：

| 边 | lag | 说明 |
|---|---:|---|
| `cast_speed_range → mold_level_range` | 0 | 拉速波动与结晶器液位波动存在强同步关系 |
| `TD_weight_range → cast_speed_range` | 0 | 中包重量波动与拉速波动存在耦合 |
| `TD_weight_range → mold_level_range` | 0 | 中包重量波动与液位波动存在耦合 |
| `TD_temp_mean → superheat` | 0 | 中包温度与过热度高度相关 |
| `cover_flux_1 → cover_flux_total` | 0 | 保护渣分量与总量存在直接关系 |

但是，将数据更新图直接并入主模型后，虽然召回略有提升，但 PR-AUC 和 F1 出现下降。因此，当前默认配置中暂时不自动启用数据更新图：

```python
paper_use_data_refined_prior = False
```

这说明当前数据更新图更适合作为图结构验证和人工审查材料，而不应直接作为强先验覆盖原有工艺图。

## 7. 实验结果

当前主要实验固定在：

```text
铸机号 = 2
奇偶流区分 = 1
```

实验结果如下：

| 方法 | PR-AUC | ROC-AUC | F1 | Precision | Recall | 说明 |
|---|---:|---:|---:|---:|---:|---|
| 旧 Full Model | 0.8250 | 0.8560 | 0.7316 | 0.8221 | 0.6591 | 复杂多尺度时序 + 强图消息融合 |
| Temporal Only | 0.8571 | 0.8951 | 0.7674 | 0.7308 | 0.8079 | 只使用时间编码，不使用图消息 |
| w/o Multi-scale Temporal | 0.8912 | 0.9019 | 0.8077 | 0.8347 | 0.7823 | 去掉复杂多尺度时序后效果提升 |
| w/o Lagged Message | 0.8238 | 0.8555 | 0.7380 | 0.8114 | 0.6768 | 不使用滞后传播 |
| w/o Relation Attention | 0.8245 | 0.8563 | 0.7296 | 0.8344 | 0.6482 | 不使用关系级注意力 |
| w/o Edge Causal Attributes | 0.8229 | 0.8555 | 0.7311 | 0.8342 | 0.6506 | 不使用边因果属性 |
| w/o Calibration | 0.8250 | 0.8560 | 0.7281 | 0.8386 | 0.6433 | 不做概率校准 |
| 当前 Full-Light | 0.8928 | - | 0.7963 | 0.7438 | 0.8567 | 当前主模型，raw+diff + 弱图消息 |
| Full-Light + Data Graph Top-80 | 0.8888 | - | 0.7915 | 0.7307 | 0.8634 | 数据图强并入，召回略升但整体下降 |
| Full-Light + Data Graph Top-30 Weak | 0.8833 | - | 0.7877 | 0.7294 | 0.8561 | 数据图弱并入，仍未优于默认 Full-Light |

当前最佳主模型为 `Full-Light`，其结果为：

```text
PR-AUC = 0.8928
F1 = 0.7963
Precision = 0.7438
Recall = 0.8567
Accuracy = 0.8485
```

与旧 Full Model 相比：

```text
F1：0.7316 → 0.7963
Recall：0.6591 → 0.8567
PR-AUC：0.8250 → 0.8928
```

这说明模型从“高精度、低召回”转向了更适合工艺预警的模式。虽然 precision 有所下降，但召回率显著提升，能够识别更多真实异常样本。

## 8. 当前结果分析

当前实验说明，模型已经具备一定异常识别能力，但仍然不能认为已经达到理想效果。主要原因如下：

第一，当前标签是规则派生标签。模型学习的是连铸过程异常规则附近的工艺状态，而不是严格意义上的真实质量缺陷。

第二，部分输入变量与标签构造规则存在重叠。这会使模型更容易学习规则本身，但会影响真实泛化能力和因果解释可信度。

第三，图结构目前仍处于候选验证阶段。数据驱动更新图主要捕捉同步相关关系，尚不能完全证明变量间存在稳定因果传播。

第四，当前实验仅固定在一条连铸线上，尚未验证跨铸机、跨流区、跨钢级和跨时间段的稳定性。

第五，当前还需要进一步分析误报和漏报样本，尤其是 FN 样本集中在哪些钢级、时间段、浇次或工况区间。

## 9. 后续优化方向

后续建议重点从以下方向继续推进：

1. 进一步人工审查 `data_refined_graph_prior.top_edges.csv`，保留工艺合理边，删除公式派生边和明显冗余边。

2. 开展边遮挡实验，验证关键边对模型性能是否真的有贡献。

3. 分析 FN 和 FP 样本，明确模型在哪些工况下失效。

4. 按钢级、浇次、时间窗口做跨工况验证，评估模型稳定性。

5. 将当前 `casting_process_fault_label` 与真实质量缺陷标签建立映射，逐步从“工艺异常识别”扩展到“质量缺陷风险预测”。

6. 在图结构上引入人工审核后的专家边，减少纯数据相关边带来的噪声。

## 10. 总结

当前工作已经形成了一个完整的数据-模型-实验闭环：首先将原始数据聚焦到连铸环节，构造连铸过程故障标签；然后建立基于 `raw + diff` 时间编码和弱图消息融合的 Full-Light 时空图模型；最后通过消融实验和数据图更新验证模型结构的有效性。

当前模型相比早期复杂 Full Model 有明显提升，尤其是召回率和 PR-AUC 明显改善。但当前模型仍然受限于标签质量、图结构可信度和跨工况泛化验证不足。后续工作的重点不应继续盲目增加模型复杂度，而应围绕标签真实性、图结构验证和错误样本分析继续优化。
