# 当前工作的六部分阶段性总结与下一步规划

## 一、研究意义与问题定位

工业过程故障诊断的核心困难不只是分类精度不足，而是异常往往以时序传播、变量耦合和控制补偿的形式出现：一个故障源可能先改变阀门、流量或温度，再通过滞后链路影响下游变量，闭环控制又会把部分异常补偿掉，使表面相关性和真实传播机理并不一致。在实际装置中，研究者通常不能随意做干预实验，不能关闭控制回路验证某条边是否是真因果，也无法完整观测操作员、控制器状态、环境扰动等隐变量。因此，本工作不应简单宣称“发现真实因果图”，也不能把相关性直接当作因果关系；更稳妥、也更有学术价值的问题是：在缺少干预数据的工业时序场景中，如何把专家知识、LLM 机理建议和统计滞后关系转化为可验证的候选机理证据，并通过可靠性评估决定它们是否允许修正强主模型，从而在提升弱类诊断能力的同时避免错误知识造成负迁移。

## 二、当前理论路线

当前模型的理论核心可以概括为“可靠性校准的机理证据融合”。给定工业过程窗口样本 `x_i`，强主模型先给出基础预测 `p_0(i)=f_0(x_i)`；专家图、LLM 规则、滞后图、正常动态残差等信息不被当作确定因果，而被构造成候选证据预测器 `p_k(i)=f_k(x_i,r_i,g_i,m_i)`，其中 `r_i` 表示正常动态残差或重构残差，`g_i` 表示工况、run、unit 或阶段上下文，`m_i` 表示候选机理边、滞后路径、符号约束或弱类规则。每个候选证据必须经过验证集收益、组间稳定性、低尾部非退化、复杂度惩罚和反事实扰动检查，形成可靠性分数或准入集合；只有当候选证据在目标弱类或目标工况中带来可验证收益，并且不会明显伤害非目标组时，才允许进入最终路由。因此，本工作的理论重点不是“堆叠一个图模块或 LLM 模块”，而是建立一套可学习、可验证、可复现的证据准入机制：机理证据可以被使用、拒绝，或从“预测器”降级为“上下文特征/弱类验证器”，这使模型具备比普通 attention、edge gate 或 graph pruning 更明确的优化目标和安全约束。

## 三、当前模型框架

当前框架可以分为四个相互约束的层次。第一层是强主辨别器，它负责从窗口特征中学习主要判别边界，在 TEP 中当前最强版本仍以严格机制增强的树模型/KIEP-GL 路线为主；在树自由对比中，我们已经测试 TCN、GRU 和 FT-Transformer 作为替代主辨别器，但它们目前还不能替代严格机制主模型。第二层是时序机理证据层，它不直接输出最终类别，而是构造短期突变、稳定漂移、振荡增强、控制补偿、阶段迁移以及滞后残差信息，并通过残差门控方式注入主预测，使图证据成为“小幅、可控、可忽略”的辅助干预，而不是覆盖主表示。第三层是多源融合图层，来源包括专家图、统计滞后检验、LLM 机理建议和验证集筛选，边上包含变量对、方向、滞后、符号、来源和可靠性；该图不被声明为真实因果图，而是“滞后机理候选图”。第四层是 ERE 可靠性评估与路由层，它学习候选证据何时比主模型更可信，并在不满足准入条件时回退到主模型或安全候选，从而把强主分类器和后处理模块的关系明确为：主模型提供锚点预测，机理证据只在局部、可验证、非退化的条件下修正锚点预测。

## 四、当前实验结果与结论

目前最强主结果来自 TEP 严格 22 类故障诊断，主模型从 `0.9122 +/- 0.0134` 提升到 `0.9549 +/- 0.0023`，提升 `+0.0428`，说明 pairwise/support-gated 机理证据对弱类故障诊断确实有效。TEP 树自由对比显示，TCN、GRU、FT-Transformer 直接替代树模型后整体效果仍明显低于严格机制模型，但在同一家族内部，残差门控滞后图均能提升 no-graph 基线：TCN 从 `0.5754` 到 `0.6034`，GRU 从 `0.6150` 到 `0.6217`，FT-Transformer 从 `0.4644` 到 `0.5136`，这证明“图证据通过残差可靠性通道进入”比“直接图平滑/全边注入”更合理。SKAB 二分类异常检测中，可靠性过滤后的动态图/LLM 路线达到 `0.8532 +/- 0.0339`，优于主基线 `0.8343 +/- 0.0341`，且反事实扰动显示边方向反转、滞后移动、随机替换目标都会降低性能，说明被接纳的边具有实际判别作用。Hydraulic 四目标任务整体接近天花板，可靠路由从 `0.9773` 到 `0.9784`，主要价值是证明安全路由不会破坏强任务，并能小幅改善最难的 valve 目标。C-MAPSS FD001-FD004 中 Fixed ERE 从 `0.7972` 到 `0.8030`，TailGuardedERE 达到 12/12 非退化，说明该机制对退化阶段诊断具有一定迁移和安全价值。总体结论是：当前结果已经能支持“可靠性校准机理证据融合”的方法贡献，但还不能直接宣称所有数据集都达到官方 SOTA。

## 五、SOTA 与 AAAI 冲击判断

从 AAAI 论文标准看，当前工作已经有比较清晰的问题意识、多数据集实验雏形和一定理论闭环，但仍处于“method-evidence”阶段，而不是可以直接宣称最终 SOTA 的阶段。TEP 是目前最适合作为主实验的数据集，因为它的 22 类故障诊断、弱类机理路径和专家知识都与本文问题高度匹配，`0.9549` 的结果也具有较强说服力；但还需要补齐 GDN-style、MTAD-GAT/Transformer 类、强时序分类器和公开论文结果的严格协议对齐。SKAB 上我们相对 USAD-style、TranAD-style、GDN-style、MTAD-GAT-style 的 matched-protocol baseline 优势明显，但由于这些还不是官方仓库级完全复现，最多能说“在当前统一协议下明显优于外部风格基线”，不能轻率写成官方 SOTA。Hydraulic 和 C-MAPSS 更适合作为泛化、安全性、非退化和近天花板场景的补充证据。若要冲击 AAAI，最低目标应是：主数据集 TEP 达到稳定 0.95 以上并完成强基线对齐；至少两个辅助公开工业数据集给出正收益或非退化证明；消融实验能证明 ERE、残差门控、LLM 弱类规则、反事实验证分别有必要；理论部分明确提出可验证的优化目标，而不是只做模块堆叠。

## 六、下一步工作规划

下一步应先冻结一个 v1 证据包，把 TEP、SKAB、Hydraulic、C-MAPSS 的结果、协议、代码入口、随机种子和指标统一到一张总表中，作为后续论文材料的基线版本。然后优先补 TEP 的强基线：在相同严格 22 类协议下加入 GDN-style、MTAD-GAT-style、Transformer/Anomaly Transformer 或更强时序图模型，确认 `0.9549` 是否仍具优势；同时补充 TEP 消融，包括无专家图、无 LLM 图、全边图、无可靠性、无残差门控、无低尾部约束、无反事实验证等。第三步是把 ERE 从经验分数进一步数学化：定义候选证据收益 `B_{g,k}`、低尾部风险 `Q_alpha(B_{g,k})`、复杂度惩罚 `C_k`、反事实下降 `CF_k` 和最终准入函数，使可靠性分数变成可训练、可复现、可检验的证据选择器。第四步是深化 LLM 的角色：LLM 不直接改最终 logit，而是为弱类生成关键变量、滞后范围、符号方向、混淆反例和可执行规则，再由验证集和反事实扰动筛选为可靠证据。第五步是在 SKAB/Hydraulic/C-MAPSS 上补齐官方协议或强基线对齐，明确每个数据集承担的论文角色。最后，论文叙事应收束为一句话：在无法直接验证真实因果的工业观测数据中，我们提出一种可靠性认证的时序机理证据融合框架，使专家知识、LLM 规则和统计滞后关系只有在通过收益、稳定性、非退化和反事实检验后才被允许修正强主模型。
